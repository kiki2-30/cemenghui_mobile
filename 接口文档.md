## 1. 用户管理模块

### 1.1 用户注册

- **POST** `/user/register`
- **请求体**（JSON）：
```json
{
  "username": "zhangsan",
  "password": "password123",
  "email": "zhangsan@example.com",
  "phone": "13800138000",
  "realName": "张三",
  "company": "科技有限公司"
}
```
- **响应示例**：
```json
{
  "code": 200,
  "message": "success!",
  "data": {
    "userId": 1,
    "username": "zhangsan",
    "email": "zhangsan@example.com",
    "phone": "13800138000",
    "realName": "张三",
    "company": "科技有限公司",
    "createTime": "2024-06-01T10:00:00",
    "updateTime": "2024-06-01T10:00:00"
  }
}
```

---

### 1.2 用户登录

- **POST** `/user/login`
- **请求体**（JSON）：
```json
{
  "username": "zhangsan",
  "password": "password123"
}
```
- **响应示例**：同上

---

### 1.3 获取用户信息

- **GET** `/user/{userId}`
- **请求示例**：
```
GET /user/1
```
- **响应示例**：同上

---

### 1.4 根据用户名获取用户信息

- **GET** `/user/username/{username}`
- **请求示例**：
```
GET /user/username/zhangsan
```
- **响应示例**：同上

---

### 1.5 更新用户信息

- **PUT** `/user/{userId}`
- **请求体**（JSON）：
```json
{
  "username": "zhangsan",
  "password": "newpassword123",
  "email": "zhangsan_new@example.com",
  "phone": "13800138001",
  "realName": "张三",
  "company": "新公司"
}
```
- **响应示例**：同注册

---

### 1.6 删除用户

- **DELETE** `/user/{userId}`
- **请求示例**：
```
DELETE /user/1
```
- **响应示例**：
```json
{
  "code": 200,
  "message": "success!",
  "data": true
}
```

---

### 1.7 检查用户名/邮箱/手机号唯一性

- **GET** `/user/check-username/{username}`
- **GET** `/user/check-email/{email}`
- **GET** `/user/check-phone/{phone}`
- **请求示例**：
```
GET /user/check-username/zhangsan
GET /user/check-email/zhangsan@example.com
GET /user/check-phone/13800138000
```
- **响应示例**：
```json
{
  "code": 200,
  "message": "success!",
  "data": true
}
```

GET /api/conferences/categories

### 2.9 会议详情管理（管理端）

- **标准定制详情**  
  - 获取：`GET /api/conferences/{conferenceId}/standard-detail`
  - 新增/编辑：`POST /api/conferences/{conferenceId}/standard-detail`
  - 更新：`PUT /api/conferences/{conferenceId}/standard-detail`
  - 删除：`DELETE /api/conferences/{conferenceId}/standard-detail`

- **技术培训详情**  
  - 获取：`GET /api/conferences/{conferenceId}/training-detail`
  - 新增/编辑：`POST /api/conferences/{conferenceId}/training-detail`
  - 更新：`PUT /api/conferences/{conferenceId}/training-detail`
  - 删除：`DELETE /api/conferences/{conferenceId}/training-detail`

- **工具研发详情**  
  - 获取：`GET /api/conferences/{conferenceId}/tool-detail`
  - 新增/编辑：`POST /api/conferences/{conferenceId}/tool-detail`
  - 更新：`PUT /api/conferences/{conferenceId}/tool-detail`
  - 删除：`DELETE /api/conferences/{conferenceId}/tool-detail`

## 3. 行业动态模块

### 3.1 行业动态列表

- **GET** `/api/news`
- **请求示例**：
```
GET /api/news?page=0&size=10&keyword=技术
```
- **响应示例**：
```json
{
  "code": 200,
  "message": "success!",
  "data": {
    "content": [
      {
        "newsId": 1,
        "title": "行业新动态",
        "summary": "摘要内容",
        "publishTime": "2024-06-01T09:00:00"
      }
    ],
    "totalElements": 100,
    "totalPages": 10,
    "currentPage": 0
  }
}
```

---

### 3.2 行业动态详情

- **GET** `/api/news/{newsId}`
- **请求示例**：
```
GET /api/news/1
```
- **响应示例**：
```json
{
  "code": 200,
  "message": "success!",
  "data": {
    "newsId": 1,
    "title": "行业新动态",
    "content": "详细内容...",
    "publishTime": "2024-06-01T09:00:00",
    "author": "管理员",
    "viewCount": 100,
    "likeCount": 20,
    "favoriteCount": 10
  }
}
```

---

### 3.3 行业动态行为（浏览、点赞、收藏等）

- **记录浏览**：`POST /api/news/{newsId}/view?userId=1`
- **点赞**：`POST /api/news/{newsId}/like?userId=1`
- **取消点赞**：`POST /api/news/{newsId}/unlike?userId=1`
- **收藏**：`POST /api/news/{newsId}/favorite?userId=1`
- **取消收藏**：`POST /api/news/{newsId}/unfavorite?userId=1`
- **查询点赞状态**：`GET /api/news/{newsId}/like-status?userId=1`
- **查询收藏状态**：`GET /api/news/{newsId}/favorite-status?userId=1`
- **获取统计数据**：`GET /api/news/{newsId}/stats`
- **批量获取统计数据**：`POST /api/news/stats`
  - **请求体**：
  ```json
  { "newsIds": [1, 2, 3] }
  ```

**响应示例**（以点赞为例）：
```json
{
  "code": 200,
  "message": "success!",
  "data": {
    "newsId": 1,
    "userId": 1,
    "liked": true
  }
}
```

---

## 4. 统一响应格式

所有接口返回如下结构：
```json
{
  "code": 200,
  "message": "success!",
  "data": { ... } // 或数组、布尔值、null等
}
```
- `code`：状态码（200成功，400参数错误，404未找到等）
- `message`：提示信息
- `data`：返回数据
